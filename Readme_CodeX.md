# プロジェクト概要
- Python + tkinter + keyboard ライブラリを使用したキーボード入力置換シーケンサーアプリ。
- 複数トリガー対応
- 出力シーケンス（hotkey / text / mouse_click）
- OSショートカットプリセット機能
- フック停止トリガー機能
- JSON保存 / 読み込み対応

# 絶対に守る設計方針
1. 構成
- 単一ファイル構成を維持する
- tkinter をGUIとして使用する
- keyboard ライブラリを使用する
- グローバルフック方式を維持する
    - トリガー検知：keyboardで行う（アプリ外でも反応させるため）
    - 記録（hotkey入力のキャプチャ）：OSショートカットはプリセット/手入力（危険なため）

2. トリガー仕様
- 通常トリガーは単キーのみ
- 停止トリガーも単キーのみ
- 停止トリガーはキャプチャ入力のみ
- 停止トリガーは suppress=True
- 停止トリガーと通常トリガーの重複は禁止

3. フック管理仕様
- suppress=True を使用
- 編集系ダイアログ表示中はフックを一時停止
- 一時停止はネスト対応（カウンタ方式）
- フック暴走を起こす設計は避ける
- UIスレッドをブロックしない

4. JSON構造（変更禁止・互換維持）
```json
{
  "triggers": [
    {
      "key": "f1",
      "label": "",
      "suppress": true,
      "actions": [
        {
          "type": "hotkey",
          "value": "ctrl+c",
          "label": ""
        }
      ]
    }
  ],
  "hotkey_presets": [
    {"label": "Win+D", "value": "windows+d"}
  ],
  "hook_stop_key": "f12"
}
```  
- 既存キーを削除しない
- 新キー追加時は後方互換を必ず保つ
- 未定義キーは無視して読み込む

追加キー（後方互換拡張）
- run_to_end: bool
- run_to_end_delay_ms: int

例
```json
{
  "key": "f1",
  "label": "",
  "suppress": true,
  "run_to_end": true,
  "run_to_end_delay_ms": 300,
  "actions": [...]
}
```

5. 表示形式（統一ルール）
- トリガー一覧
    01. key: label
- 出力シーケンス
    01. [type] value: label
- プリセット一覧
    01. value: label
- ラベルは空でも可
- ラベル空の場合は「:」を表示しない

6. 入力仕様
- hotkey は validate_hotkey() で事前検証
- 不正値は messagebox で通知し、アプリを止めない
- OSショートカットはプリセット方式を推奨
- マウスクリックは (x, y, button, clicks) 形式

7. コード提案ルール
- 必ず差分形式で提示する
- 既存レイアウト（grid行）を不用意に変更しない
- 既存機能を壊さない最小差分を優先する
- 既存の命名・設計を尊重する

8. 安全設計優先
- フックでUIを閉じ込めない
- Alt+Tab等の危険ショートカットに配慮
- suppressの副作用を考慮する
- try/exceptでkeyboard例外を吸収する

9. パフォーマンス方針
- 無駄な再フックを避ける
- フック登録/解除は必要最小限
- UIスレッドで重い処理を行わない

10. UI構成（前提）
- FullView / CompactView の2画面構成
- pack_forget による表示切替
- CompactView は最小UI（フック操作 / トリガー一覧 / 表示設定）
- FullView はすべての編集機能を含む
- Listbox（トリガー / シーケンス / プリセット）はスクロール対応

11. 連続実行機能（run_to_end）
- トリガーごとに連続実行設定を持つ
- チェックボックス「連続実行」で有効化
- トリガー押下で現在のシーケンス位置から最後まで実行
- 最後まで実行したら停止し、次回は先頭から開始
- 連続実行中に同トリガー押下で一時停止
- 一時停止中に同トリガー押下で再開
- 連続実行中は他トリガーの実行を抑止
- フック停止トリガーはこの制御の対象外

12. 連続実行間隔
- トリガーごとに実行間隔(ms)を設定
- デフォルト値は 300ms
- UIは「連続実行」チェックボックス直下に配置
- JSONキー
  - run_to_end: bool
  - run_to_end_delay_ms: int

13. UI同期ルール
- トリガー選択変更時は関連UIを同期する
- run_to_end関連UIの同期関数は `_sync_run_to_end_ui()` に統一
- suppress / run_to_end / delay はすべてトリガーごとの設定として扱う
