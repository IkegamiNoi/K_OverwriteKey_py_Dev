# タスク2 指示書

## 物理キーサブ表示 + 表示ON/OFF設定 + 予約キー表示の見やすさ改善

### 目的

キーボードUIで

* 主表示: 実行結果
* サブ表示: 物理キー名

を切り替え可能にし、表示の分かりやすさを上げる。
あわせて、予約キー表示を主表示・サブ表示の両面で見やすくする。

---

### 今回やってほしいこと

#### 1. キーボードUIにサブ表示を追加

各キーキャップで、主表示とは別に **サブ表示用テキスト** を描画できるようにする。

用途:

* 主表示: 実行結果
* サブ表示: 物理キー名

例:

* triggerキー → 主表示 `01`、サブ表示 `Q`
* keymapキー → 主表示 `7`、サブ表示 `Q`
* stopキー → 主表示 `STOP`、サブ表示 `F12`

---

#### 2. サブ表示は設定でON/OFFできるようにする

メニューバーにチェック項目を追加する。

文言:

* `物理キー名を表示`

要件:

* ON のときだけサブ表示を描画
* OFF のときは従来どおり主表示のみ
* 設定状態は config.json に保存
* 再起動時に復元

保存キー名は既存命名に合わせてよいが、例として:

* `keyboard_show_physical_key_labels`

---

#### 3. 物理キー名は表示用に整形してよい

サブ表示の物理キー名は、内部キー名そのままではなく、必要なら短く整形してよい。

例:

* `shift_l` → `Shift`
* `ctrl_l` → `Ctrl`
* `windows_l` → `Win`
* `page up` → `PgUp`

ただし今回は最小差分優先なので、既存の `label` を流用できるならそれでもよい。

---

#### 4. 予約キーの主表示/サブ表示ルール

予約キーは今回の色分けに加えて、表示でも分かりやすくする。

主表示:

* 停止キー → `STOP`
* 通常モード切替キー → `MODE`
* キーマップ切替キー → `KEYMAP`

サブ表示:

* 物理キー表示ON時のみ、実際の物理キー名を表示

---

### 実装制約

* 既存の render() 全体を作り直さない
* `key_items` 構造を大きく壊さない
* 最小差分で、主表示とサブ表示のテキスト追加だけで対応する
* フック処理には触れない
* 色分けロジックは今回変更しない

---

### 実装の方向性

以下のような対応でよい。

* 既存の1つの text 描画を

  * `main_text`
  * `sub_text`
    の2つに分ける
* サブ表示は小さめフォント
* サブ表示はキー下側または左下寄せなど、主表示と重ならない位置
* 表示OFF時は `sub_text` を描画しない、または空文字にする

---

### config 追加

config.json に以下を追加する。

* `keyboard_show_physical_key_labels`

保存対象:

* メニューのチェック変更時に即保存

---

### UI反映

以下の操作時に即再描画されること。

* メニューの表示ON/OFF切替
* レイアウト変更
* トリガー変更
* キーマップ変更
* モード変更

---

### 禁止事項

* レイアウト定義の全面変更
* キー状態判定ロジックの変更
* InputRouter / ActionExecutor への変更
* 既存の色分けルール変更

---

### 出力形式

* 変更箇所のみ差分形式
* コメントは必要最小限
* 動作確認項目も提示すること

---

### 動作確認項目

1. メニューバーの `物理キー名を表示` ON/OFF でサブ表示が切り替わる
2. OFF のとき従来どおり主表示のみになる
3. ON のとき trigger / keymap / stop / mode / keymap_toggle でサブ表示が出る
4. レイアウト変更後もサブ表示が崩れない
5. config 保存後、再起動して設定が復元される
6. 既存の色分けが壊れていない

このタスクの確認が終わったら、次は **不要コード整理（modifiers 削除 / HookCoordinator 整理 / 命名修正）** に進む想定です。
