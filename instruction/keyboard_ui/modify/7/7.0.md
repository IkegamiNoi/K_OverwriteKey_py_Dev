# タスク7 指示書

## keymap ラベル編集 + 一覧ダブルクリックで active 化

---

## 目的

現在の keymap 管理UIでは、

* `keymap_1`, `keymap_2` のような内部ID中心で見える
* active 化はボタン操作のみ

となっている。

このタスクでは、実運用しやすくするために以下を追加する。

* keymap の表示ラベル編集
* keymap 一覧のダブルクリックで即 active 化

---

## 今回やること

### 1. keymap の表示ラベル編集を追加

keymap は内部IDとは別に、**表示用ラベル**を持てるようにする。

用途:

* `keymap_1` → `numpad`
* `keymap_2` → `left_hand`
* `keymap_3` → `game`

のように、人が分かりやすい名前を付けられるようにする。

---

### 2. keymap 一覧をラベル優先表示にする

Listbox には

* ラベルがある場合 → ラベルを表示
* ラベルが空または未設定 → id を表示

のルールで出す。

必要なら補助的に ID も見える形にしてよいが、最小実装ならラベル優先で十分。

例:

```text id="u8x6zy"
01. numpad
02. left_hand
03. game
```

ラベル未設定なら:

```text id="tl6t22"
01. keymap_1
```

---

### 3. ラベル編集ボタンを追加

keymap 管理UIに新しいボタンを追加する。

文言例:

* `名前変更`

動作:

* 現在選択中の keymap を対象にする
* simpledialog などでラベル文字列を入力させる
* 確定したら保存
* 一覧表示を即更新
* キーマップ切替UIにも反映

---

### 4. ラベル保存のルール

内部IDは変更しない。
変更するのは **label のみ**。

つまり:

* `id`: 内部識別子
* `label`: 表示名

と分ける。

理由:

* 既存JSON互換を壊しにくい
* 参照関係を壊さない
* 後から編集しやすい

---

### 5. ダブルクリックで active 化

keymap 一覧の Listbox に

* `<Double-Button-1>`

を bind し、ダブルクリックした項目を active keymap にする。

要件:

* 既存の `選択` ボタンと同じ処理を使ってよい
* active 化後、一覧表示とキーマップ切替UIを更新
* キーボードUIが開いていれば必要に応じて反映

---

## データ構造

既存の keymap データに **label** を追加できるようにする。
ただし後方互換を壊さないこと。

例:

```json
{
  "id": "keymap_1",
  "label": "numpad",
  "mapping": {
    "q": "7"
  }
}
```

もし現在の keymap 構造が辞書ベースなら、それに合わせて最小差分で実装してよい。
重要なのは **label が任意項目として保存できること**。

---

## 既存データとの互換性

既存の keymap に `label` が無い場合でも正常動作すること。

ルール:

* `label` 無し → id を表示
* 保存時に `label` を追加できる

---

## UI仕様

### keymap 管理UIに追加するもの

* `名前変更` ボタン

### 一覧表示

* ラベル優先表示
* active が分かる状態を維持

### ダブルクリック

* 即 active 化

---

## UI更新タイミング

以下の操作後に更新すること。

* keymap 追加
* keymap 削除
* keymap active 化
* ラベル変更
* JSON読込
* 新規作成

---

## 実装方針

以下のような関数を追加してよい。

* `_rename_keymap_label()`
* `_on_keymap_list_double_click()`

既存の

* `_select_keymap()`
* `_refresh_keymap_list_ui()`
* `_refresh_keymap_switch_ui()`

などをなるべく再利用してほしい。

---

## 禁止事項

* keymap の内部ID変更機能を入れない
* JSON構造を大きく作り直さない
* フック処理に触れない
* キーボードUIの描画ロジックには触れない
* 大規模なUI再配置をしない

---

## 出力形式

* 追加した保存項目とUI部品を簡潔に説明
* 動作確認項目も提示すること

---

## 動作確認項目

### ラベル編集

1. keymap を選択して `名前変更` ができる
2. ラベル変更後、一覧表示が即更新される
3. キーマップ切替UIにも反映される
4. ラベル未設定 keymap は id 表示のままで動く
5. JSON保存→再読込でラベルが復元される

### ダブルクリック

6. 一覧をダブルクリックするとその keymap が active になる
7. `選択` ボタンと同じ結果になる
8. active 化後、一覧表示が更新される
9. キーマップ切替UIにも反映される

### 既存影響

10. keymap 追加・削除が壊れていない
11. active keymap 削除時のフォールバックが壊れていない
12. キーボードUIでの keymap 編集保存が壊れていない

---

* keymap にラベルを付けられる
* 一覧はラベル優先で表示される
* ダブルクリックで active 化できる
* 既存機能が壊れていない
