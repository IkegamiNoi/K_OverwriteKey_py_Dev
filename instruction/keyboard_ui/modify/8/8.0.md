## タスク8

### キーマップ切替キー群（直接指定方式）の追加

---

## 目的

現在の巡回式キーマップ切替に加えて、

👉 **キーごとに特定の keymap を直接選択できる機能**

を追加する。

---

## 仕様

### 新機能

キー入力で keymap を直接切替：

```text
1 → numpad
2 → default
3 → left_hand
```

---

## データ構造

### 追加

```json
"keymap_switch_keys": {
  "1": "keymap_1",
  "2": "keymap_2"
}
```

---

## 実装内容

### 1. KeymapService に追加

#### データ保持

```python
self.data["keymap_switch_keys"]
```

#### メソッド

```python
def set_keymap_switch_key(self, key: str, keymap_id: str)
def remove_keymap_switch_key(self, key: str)
def get_keymap_by_switch_key(self, key: str) -> Optional[str]
```

---

### 2. InputRouter に追加

キー入力時：

```python
# 最優先でチェック
km_id = keymap_service.get_keymap_by_switch_key(key)

if km_id:
    keymap_service.set_active_keymap_id(km_id)
    return  # 他処理しない
```

---

### 3. 優先順位

```text
停止キー
> モード切替
> キーマップ切替キー（新）
> トリガー
> キーマップ
```

---

### 4. UI追加

## keymap管理UIに追加

### 表示

```text
切替キー設定:

1 → numpad
2 → default
```

---

### 操作

#### 追加ボタン

* キー入力待ち状態へ
* 押したキーを取得
* 選択中 keymap に割り当て

#### 削除ボタン

* 選択キー削除

---

## 表示更新

* `_refresh_keymap_switch_ui()` に統合
* 変更時即更新

---

### 5. 重複禁止

以下は禁止：

* 停止キー
* モード切替キー
* 巡回切替キー
* 既存トリガーキー

👉 重複時はエラー表示（messagebox）

---

### 6. JSON保存

* config.json に保存
* 読込対応

* 既存巡回式を壊さない
