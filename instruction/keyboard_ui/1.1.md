キーボードUIの配列定義まわりを整理したい。
現在は英語配列テンキーレス相当の `KEY_SPECS` をコード内固定で持っているが、今後以下に対応したい。

* 日本語配列
* 英語配列
* 80% キーボード
* ハーフキーボード
* 将来的な独自レイアウト追加

そのため、**描画ロジックとキーボード配列定義を分離**して、
**配列定義を差し替え可能な構造**に変更してほしい。

## 今回やってほしいこと

### 1. `KEY_SPECS` 固定依存をやめる

現在の描画コードが `KEY_SPECS` 固定前提ならやめて、
**「KeySpec の一覧を受け取って描画するだけ」** の構造にしてほしい。

つまり描画側は

* どの配列か
* 英語配列か日本語配列か
* TKLかフルか

を知らず、**受け取った配列定義をそのまま描画するだけ**にする。

---

### 2. デフォルト内蔵レイアウトを用意する

今ある英語配列テンキーレス相当の `KEY_SPECS` は削除せず、
**デフォルト内蔵レイアウト**として残してよい。

ただし単なる定数ではなく、
将来的に複数レイアウトを持てる形に整理してほしい。

イメージ:

* `DEFAULT_LAYOUT_ID = "us_tkl"`
* `BUILTIN_LAYOUTS = { "us_tkl": ... }`

のように、**レイアウトIDで引ける形**にしてほしい。

---

### 3. 外部JSONからレイアウト定義を読める構造を追加する

今すぐ複数JSONを大量に作らなくてよいが、
**外部JSONレイアウトを後から追加できる構造**にしてほしい。

今回は最低限、

* JSONから1レイアウトを読み込む関数
* そのJSONを `KeySpec` 一覧へ変換する処理

まで入れてほしい。

外部JSONが無い場合や読み込み失敗時は、
**内蔵の英語配列テンキーレスを使う** フォールバックにしてほしい。

---

### 4. キーボード描画は「現在のレイアウト定義」を使う

KeyboardView / KeyboardWindow 側は
固定 `KEY_SPECS` ではなく、**現在選択中のレイアウト定義**を使って描画するように変更してほしい。

将来的に

* 英語配列
* 日本語配列
* 80%
* ハーフ

へ差し替えても、描画コードの修正が不要な構造にしたい。

---

### 5. 今回は新レイアウト追加までは不要

今回は **仕組み作りだけ** が目的。

なので今回必須なのは以下まで。

* 現在の英語配列TKLを内蔵レイアウト化
* 外部JSON読込対応
* 描画側の固定依存排除

以下は **まだ不要**。

* 日本語配列レイアウトJSONの完成
* 80%レイアウト実装
* ハーフキーボード実装
* レイアウト切替UI追加

---

## レイアウトJSONの想定構造

JSONは最低限、以下のような構造を想定してほしい。

```json
{
  "layout_id": "us_tkl",
  "display_name": "US TKL",
  "keys": [
    {"id": "esc", "label": "Esc", "x": 0.0, "y": 0.0, "w": 1.0, "h": 1.0},
    {"id": "f1", "label": "F1", "x": 2.0, "y": 0.0, "w": 1.0, "h": 1.0}
  ]
}
```

### 各キーの項目

* `id`: 内部識別子
* `label`: 表示文字
* `x`, `y`: unit基準座標
* `w`, `h`: unit基準サイズ

---

## `id` についての注意

記号キーは将来の日本語配列対応で揺れやすいので、
可能なら **内部IDと表示ラベルを分ける** 前提で整理してほしい。

たとえば `"\"` のような記号をそのままIDにするより、
可能なら `"backslash"` のような安定した識別子の方が望ましい。
ただし今回は既存コードへの影響が大きいなら、**今のIDを維持してもよい**。
この点は最小差分優先で進めてほしい。

---

## 既存構造への配慮

既存機能を壊さないこと。

特に以下は維持してほしい。

* 現在のキーボードUI表示
* unitベース描画
* リサイズ時の再描画
* 既存のキーIDベース処理（影響がある場合は最小差分）

---

## 実装方針

以下の順で整理してほしい。

1. `KeySpec` 一覧を返す「レイアウト取得層」を追加
2. 現在の `KEY_SPECS` を内蔵レイアウトへ移行
3. JSON読込関数追加
4. 描画コードを「現在レイアウト参照」に変更
5. フォールバックを英語配列TKLに設定

---

## 禁止事項

* 既存のキーボードUI全体を大きく作り直さない
* 今回の段階で80%/ハーフ/JISの完成実装まで広げない
* 既存の入力処理やフック処理には触らない
* 既存の grid / window 構成を不用意に変えない
