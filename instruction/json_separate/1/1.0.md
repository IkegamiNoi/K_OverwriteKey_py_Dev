# 🛠 タスク1 実装指示書

## 🎯 目的

* `settings` → `config` への移行
* 新パス導入
* 旧パスフォールバック対応

---

## ⚠️ 絶対条件

* フック処理を変更しない
* UIを壊さない
* JSON構造はまだ変えない
* 最小差分

---

## 🔧 実装内容

---

### ① configルートの導入

App初期化で以下を定義：

```python
self.config_root = os.path.join(base_dir, "config")
self.user_root = os.path.join(self.config_root, "user")
```

---

### ② パス定義を変更

#### 変更前（例）

```python
self.startup_path = ...
self.config_path = ...
self.keyboard_layouts_dir = ...
```

---

#### 変更後

```python
self.startup_path = self._resolve_startup_path()
self.config_path = self._resolve_config_path()
self.keyboard_layouts_dir = self._resolve_keylayout_dir()
```

---

### ③ パス解決ヘルパー追加

```python
def _resolve_startup_path(self):
    new_path = os.path.join(self.config_root, "config.json")
    old_path = os.path.join(base_dir, "settings", "startup.json")
    return new_path if os.path.exists(new_path) else old_path
```

---

```python
def _resolve_config_path(self):
    new_path = os.path.join(self.user_root, "config.json")
    old_path = os.path.join(base_dir, "settings", "config.json")
    return new_path if os.path.exists(new_path) else old_path
```

---

```python
def _resolve_keylayout_dir(self):
    new_path = os.path.join(self.user_root, "keylayout")
    old_path = os.path.join(base_dir, "keylayout")
    return new_path if os.path.exists(new_path) else old_path
```

---

## ④ 保存先は新パス優先

* 保存時は必ず `config/user/` に保存
* 旧 `settings/` には保存しない

---

## ⑤ ディレクトリ自動生成

起動時に：

```python
os.makedirs(self.user_root, exist_ok=True)
```

---

## ⑥ 既存ロジックは変更しない

* `ensure_config_compatibility()` はそのまま
* trigger / keymap / UI 更新処理は触らない

---

# 🧪 動作確認

* settingsしかない状態で起動できる
* configフォルダが自動生成される
* 保存するとconfig/userに保存される
* keylayoutが旧フォルダでも読める
* 既存機能が壊れていない
