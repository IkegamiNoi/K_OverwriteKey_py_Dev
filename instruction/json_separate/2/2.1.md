# タスク2.1 指示書

**題名:** 分離JSONのサンプル作成と最小動作確認

## 目的

タスク2で実装した**分離JSON読込**について、
実際に読込経路を通せる最小構成のサンプルJSONを作成し、
**最低限の動作確認ができる状態**を作る。

このタスクの目的は、分離保存の実装ではなく、
**分離読込の実データ確認**である。

---

## このタスクの位置づけ

* タスク2の実装後
* 旧単一JSONの読込確認は済んでいる
* ただし、分離JSONを実際に置いた試験は未実施
* そのため、まずは**最小サンプルを用意して分離読込経路を確認する**

---

## 絶対条件

* 既存コードを壊さない
* UIレイアウトを変更しない
* フック処理を変更しない
* 既存の単一JSONフォールバックは壊さない
* サンプルは**最小構成**にする
* 既存の手動配置済みファイルを不用意に上書きしない

---

## 作業内容

### 1. 分離JSONの最小サンプルファイルを作成する

以下のファイルを **存在しない場合のみ** 作成すること。
既に存在する場合は上書きしない。

---

### 作成対象

#### `config/config.json`

```json
{
  "keymap_set_path": "user/keymap_sets/default.json",
  "ui_font_delta_pt": 0,
  "last_used_directory": ""
}
```

---

#### `config/user/keymap_sets/default.json`

```json
{
  "trigger_set_path": "user/trigger_sets/default.json",
  "hotkey_presets_path": "user/hotkey_presets/default.json",
  "active_keymap_path": "user/keymaps/default.json",
  "keymaps": [
    {
      "path": "user/keymaps/default.json",
      "switch_key": "f6"
    }
  ],
  "hook_stop_key": "f12",
  "hook_toggle_key": "",
  "keyboard_layout": "us_tkl",
  "keyboard_show_physical_key_labels": false,
  "debug_jis_special_key_events": false,
  "external_keyboard_layouts": []
}
```

---

#### `config/user/keymaps/default.json`

```json
{
  "label": "Default",
  "mappings": {}
}
```

---

#### `config/user/trigger_sets/default.json`

```json
{
  "triggers": []
}
```

---

#### `config/user/hotkey_presets/default.json`

```json
{
  "hotkey_presets": []
}
```

---

### 2. 必要なディレクトリを作成する

以下のディレクトリが無ければ作成する。

* `config/`
* `config/user/`
* `config/user/keymap_sets/`
* `config/user/keymaps/`
* `config/user/trigger_sets/`
* `config/user/hotkey_presets/`

---

### 3. サンプル作成はコード実装ではなく、リポジトリ上のファイル追加で行う

このタスクでは、アプリ実行時にサンプルを自動生成する処理は追加しないこと。

つまり、

* 起動時自動生成
* 不足時自動生成
* サンプル生成用helper追加

は**やらない**。

あくまで、開発・確認用として
**静的ファイルを追加するだけ**にする。

---

### 4. 最小動作確認を行う

サンプル作成後、以下の観点で確認すること。

#### 確認項目

1. `config/config.json` から `keymap_set_path` が読まれる
2. `keymap_set.json` から `default.json` 群が読まれる
3. 分離構成から起動してもエラーで落ちない
4. `keymaps` に `default` が1件読み込まれる
5. `switch_key: "f6"` がランタイムの `keymap_switch_keys` に反映される
6. trigger / hotkey_presets が空でも正常起動する
7. 旧単一JSON読込フォールバックを壊していない

---

## 実装方針

### 1. 既存ファイルを上書きしない

このタスクは試験用サンプルの追加が目的。
すでに同名ファイルがある場合は、既存内容を尊重すること。

必要なら、既存ファイルがある場合は

* 作成をスキップ
* 差分説明でその旨を明記

でよい。

---

### 2. サンプル内容は最小限にする

今回の目的は、分離読込経路を通すこと。
そのため、以下のような追加データは不要。

* trigger の実サンプル追加
* hotkey action の複雑な例
* 複数 keymap
* external keyboard layout の実ファイル
* UI用の過剰な設定値

まずは**空でもよいから読めること**を優先する。

---

### 3. 既存コード変更は必要最小限

基本は**ファイル追加のみ**でよい。
もし動作確認のために軽微な修正が必要なら差分で示してよいが、
不要ならコードは変更しないこと。

---

## 変更対象の目安

* `config/config.json`
* `config/user/keymap_sets/default.json`
* `config/user/keymaps/default.json`
* `config/user/trigger_sets/default.json`
* `config/user/hotkey_presets/default.json`

必要ならディレクトリ追加のみ。

---

## 動作確認結果として報告してほしいこと

CodeX の返答では、少なくとも以下を報告すること。

1. 作成したファイル一覧
2. 既存ファイルがありスキップしたものがあるか
3. 分離構成での起動確認結果
4. 旧単一JSONフォールバックに影響がないか
5. もし未確認項目があれば、その理由

---

## このタスクでやらないこと

* 分離保存の実装
* 個別保存
* keymap_set のUI切替
* trigger や preset の複雑なサンプル追加
* 自動サンプル生成機能
* 旧 settings からの移行ロジック追加

---

## 補足

今回のサンプルは**開発確認用**であり、本番データ移行ではない。
そのため、

* 最小構成
* 静的追加
* 上書きしない

を守ること。
