# タスク4.1 指示書（改訂版）

**題名:** パス制約の緩和・命名整理・Import/Export統合（UI含む）

---

## 🎯 目的

タスク4で導入した split 本流 + Import/Export 構成を整理し、

* 不要なパス制限の撤廃
* パス管理ルールの統一（相対/絶対）
* 命名のズレ修正
* UI表記の整合

を行う。

---

## ❗ 絶対条件

* フック処理を変更しない
* UIレイアウト（grid）は変更しない
* 既存機能を壊さない
* 最小差分で実装する

---

# 🧠 設計確定事項

## 1. パス保存ルール（重要）

### 保存時

* config_root 配下 → 相対パス
* それ以外 → 絶対パス

### 読込時

* 絶対パス → そのまま使用
* 相対パス → config_root 基準で解決

---

## 2. 読込制限の撤廃

以下は **すべて制限を外す**

* `load_from()` の config配下チェック → 削除
* `save_as()` の config配下チェック → 削除

---

## 3. 起動設定

* `config/config.json` の場所は固定
* `keymap_set_path` は自由（相対/絶対）

---

## 4. 保存動作

* `save` は現在の `self.keymap_set_path` に保存
* 外部ファイルでもそのまま上書きする

---

## ③ パス正規化処理の追加/整理

### 新規または統一関数

```python
def _to_config_relative_or_absolute(path, config_root):
    if path is under config_root:
        return relative path
    else:
        return absolute path
```

これを以下で使用：

* 保存時
* startup更新時
