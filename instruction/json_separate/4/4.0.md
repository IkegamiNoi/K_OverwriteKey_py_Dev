# タスク4 指示書（改訂版）

**題名:** 分離構成の本流化 + Import/Export機能追加（UI含む）

---

## 🎯 目的

分離構成（split JSON）を本流に統一し、
旧単一JSON形式を **Import / Export 機能として独立**させる。

---

## 🧭 到達目標

### 本流（split構成）

* 起動時読込：分離構成のみ
* 保存：分離構成のみ
* 別名保存：分離構成のみ
* 読込：`keymap_set.json`

### 互換機能

* Import：単一JSON → `self.data`
* Export：`self.data` → 単一JSON

---

## ❗ 絶対条件

* フック処理を変更しない
* UIレイアウトを壊さない（追加のみ）
* ランタイム構造を維持
* 最小差分で実装
* split構成の動作を壊さない

---

# 🧠 設計

## 1. 本流と互換の完全分離

| 機能     | 形式     |
| ------ | ------ |
| 保存     | split  |
| 読込     | split  |
| 起動     | split  |
| Import | 単一JSON |
| Export | 単一JSON |

---

## 2. 読込仕様（確定）

* `keymap_set.json` を選択
* split読込
* UI反映
* dirty=False
* 起動設定は変更しない

---

## 3. Import仕様（確定）

* 単一JSON読込
* `ensure_config_compatibility()` を通す
* `self.data` 更新
* UI更新
* **dirty=True**
* 保存はしない

---

## 4. Export仕様（確定）

* `self.data` を単一JSONで保存
* 内部キー除去
* split構成は変更しない

---
